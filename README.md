# Scripts-pipeline-SVs

A bioinformatics pipeline for processing VCF files generated by three structural variant (SV) callers in whole-genome sequencing (WGS) samples. The pipeline consolidates all detected variants into a single BED file for downstream analysis.

---

## How to run this pipeline?

The files for the sample you wish to analyze must be stored in the same directory, organized as follows:

**Example directory structure:**

```
/0032_0032/
├── Manta/
│     Manta_0032.vcf
├── delly/
│     delly_0032.bcf
└── GRIDSS/
      GRIDSS_0032.vcf
```

Once organized, simply execute:

```bash
./pipe_SV.sh path/to/directory/0032_0032
```

> [!IMPORTANT]
> Do not include a trailing `/` when specifying the directory path.

## Step-by-step overview

### 1. Filtering and preparation

- Directories are created to filter variants with `PASS` quality.
- VCF files are converted into BED format.

### 2. Processing per SV type

- Scripts process formatted files for each caller and SV type.
- SVs are separated by genotype, then sorted and grouped by coordinates.
- Specific tools handle certain SVs (e.g., `bedtools merge` for inversions and deletions).

### 3. Annotation and consolidation

- Necessary annotations are added.
- All results are combined into a single final file with a specific format (see final output header below).
- The final file contains all variants grouped by coordinates, with information on overlaps across callers to create a consensus view.

## Final output format

| Column | Description |
|--------|-------------|
| `#chr` | Chromosome |
| `start` | Start coordinate of the merged SV region |
| `end` | End coordinate of the merged SV region |
| `ID_PROGRAM` | Comma-separated list of individual variant IDs from all supporting callers |
| `SVTYPE` | Structural variant type (DEL, INS, DUP, INV, TRA) |
| `merged_len` | Length of the merged SV region |
| `N_programs` | Number of distinct callers supporting this variant |
| `Programs` | Supporting callers: G = GRIDSS, D = delly, M = Manta |
| `GT_MD` | Genotype (0/1 heterozygous, 1/1 homozygous) |
| `G_overlap` | Per-variant GRIDSS overlap ID and percentage |
| `G_Total_overlap` | Total GRIDSS overlap percentage for this region |
| `G_overlap_INFO` | Summary: `Total_overlap=per_variant_overlaps` (GRIDSS) |
| `G_coord` | GRIDSS variant coordinates (`chr:start-end`) |
| `GRIDSS_VAF` | GRIDSS variant allele frequency (VAF) |
| `D_overlap` | Per-variant delly overlap ID and percentage |
| `D_Total_overlap` | Total delly overlap percentage for this region |
| `D_overlap_INFO` | Summary: `Total_overlap=per_variant_overlaps` (delly) |
| `D_coord` | delly variant coordinates (`chr:start-end`) |
| `M_overlap` | Per-variant Manta overlap ID and percentage |
| `M_Total_overlap` | Total Manta overlap percentage for this region |
| `M_overlap_INFO` | Summary: `Total_overlap=per_variant_overlaps` (Manta) |
| `M_coord` | Manta variant coordinates (`chr:start-end`) |

## Script reference

| Script | Language | Module | Purpose |
|--------|----------|--------|---------|
| `pipe_SV.sh` | Bash | Main | Master pipeline script; orchestrates all processing and merge steps |
| `procesado_inicial.sh` | Bash | Processing | BCF-to-VCF conversion, chrM removal, PASS/No_PASS splitting |
| `separar_portiposM.py` | Python | Processing | Splits Manta VCF files by SVTYPE |
| `vcf2bed.sh` | Bash | Processing | Converts all PASS VCF files to BED format using `vcf2bed` |
| `delly_processing.py` | Python | Processing | Reformats delly BED files; fixes coordinates and assigns variant IDs |
| `separar_BND_Manta.py` | Python | Processing | Classifies Manta BND records as INV or TRA by chromosome comparison |
| `pre_processing_Manta2.py` | Python | Processing | Reformats Manta BED files; fixes coordinates and assigns variant IDs |
| `script_GridssVariantAnnotation.R` | R | Processing | Annotates GRIDSS BND pairs with SV type, coordinates, and length |
| `pre_procesado_GRIDSS.py` | Python | Processing | Updates GRIDSS BED with SV annotations from R output; computes VAF |
| `separar_portiposG.py` | Python | Processing | Splits processed GRIDSS BED file by SVTYPE; assigns variant IDs |
| `merge.sh` | Bash | Merge | Coordinates all merge steps; concatenates final output |
| `split_gt.py` | Python | Merge | Splits BED files into homozygous and heterozygous variant files |
| `del_merge.py` | Python | Merge | Merges deletions from all callers; computes overlaps and per-caller info |
| `inv_merge.py` | Python | Merge | Merges inversions from all callers; handles INV1/INV2 coordinate logic |
| `dup_merge.py` | Python | Merge | Merges duplications from all callers; groups by coordinate |
| `ins_merge.py` | Python | Merge | Merges insertions from all callers; uses ALT sequence length for overlap |
| `tra_merge.py` | Python | Merge | Merges translocations from all callers; groups by coordinate and CHR2 |
